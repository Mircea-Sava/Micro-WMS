# Warehouse.xlsm - Complete Structure Reference

## Workbook Overview

| Sheet | Index | Purpose |
|-------|-------|---------|
| Form | 1 | User interface for entering pick/putaway orders |
| Map_Grid | 2 | Visual warehouse map (A1:AO33) with A* pathfinding overlay |
| Map_Helper | 3 | Backend database for bin locations, inventory state, and SKU registry |
| Transaction_History | 4 | Permanent log of all confirmed operations |

All VBA code lives in a single module: **Module1** (1089 lines).

---

## Sheet 1: Form

**Purpose**: Primary input form where users enter pick/putaway orders before confirming.

| Column | Header | Data Type | Description |
|--------|--------|-----------|-------------|
| A | SKU | Text | Item identifier (e.g., "AAA", "BBB") |
| B | Qty | Number | Quantity of units to pick or put away |
| C | Operation | Text | Must be "PICK" or "PUTAWAY" |
| D | Location Assigned | Text | Bin name (e.g., "A1-R1-L1-A"). Left blank for auto-assignment. Comma-separated for multi-stop. |

**UI Elements**: 3 Shapes (buttons) on the sheet.

**Usage**: Fill rows 2+ with order lines, then run `ConfirmOperations`. If Location is left blank on a PUTAWAY, the system auto-assigns a bin. If blank on a PICK, the system finds a bin containing that SKU.

---

## Sheet 2: Map_Grid

**Purpose**: Visual 2D representation of the warehouse floor, drawn to scale from an AutoCAD drawing.

**Grid Range**: A1:AO33 (41 columns x 33 rows)

| Cell Value | Meaning |
|------------|---------|
| `1` | Wall / Obstacle (impassable by A* pathfinding) |
| `2` | Dock / Start Point (worker entry, used as pathfinding origin) |
| Bin name (e.g., `A1-R1-L1-A`) | A storage location on the floor |
| Empty | Open floor (walkable) |

**Scale**: Each cell = 0.3848m (stored as `TILE_SCALE` constant in VBA). Derived from the AutoCAD drawing ratio: 620" x 500" mapped onto 41 x 33 cells.

**Visual Elements Generated by VBA**:
- **Path lines**: Colored connectors drawn by `DrawGPSPath`. Color gradient goes green-to-red across the route to show progress.
- **Target markers**: Shapes placed at bin locations by `DrawTargetMarker`:
  - Oval = Level 1 bin
  - Triangle = Level 2 bin
  - Rectangle = Other levels
  - Red fill = PICK operation
  - Green fill = PUTAWAY operation
- **SKU labels**: Tiny text boxes next to markers showing the SKU name.
- **Stats box**: Dark rounded rectangle in the top-right area showing "TOTAL: X.Xm" total walking distance.

**Cell AQ1**: Stores the calculated total route distance (meters) after `GenerateMapPath` runs.

All generated shapes are named `PathLine_Generated` or `PathLabel_Generated` so `ClearMap` can remove them without affecting the base map.

## Sheet 3: Map_Helper

**Purpose**: Backend database combining the bin registry, coordinate lookup, distance/rank analysis, SKU master list, and live inventory state.

### SKU Master Registry (Columns XEU:XEV)

| Column | Header | Data Type | Description |
|--------|--------|-----------|-------------|
| XEU | SKU | Text | Registered SKU identifier |
| XEV | Capacity | Number | Maximum units this SKU can hold per bin (default 50) |

New SKUs are registered via `RegisterSKU` when first encountered during `ConfirmOperations`. The user is prompted to enter a capacity.

### Bin Database (Columns XEX:XFD)

| Column | Header | Data Type | Source | Description |
|--------|--------|-----------|--------|-------------|
| XEX | Bin Name | Text | Formula (SORT/UNIQUE) | Unique bin names extracted from Map_Grid cells |
| XEY | Grid X | Number | Formula (MAP/LAMBDA) | Column number of the bin on Map_Grid |
| XEZ | Grid Y | Number | Formula (MAP/LAMBDA) | Row number of the bin on Map_Grid |
| XFA | Distance (m) | Number | `Analyze_Map_Locations` | Round-trip A* distance from dock to bin and back |
| XFB | ABC Rank | Text (A/B/C) | `Analyze_Map_Locations` | Classification based on distance percentiles |
| XFC | Current SKU | Text | `RebuildInventoryState` | What SKU is currently stored in this bin ("EMPTY" if vacant) |
| XFD | Current Qty | Number | `RebuildInventoryState` | How many units are currently in this bin |

**Note**: Column XEW is intentionally skipped/unused.

**Dynamic Formulas** (set up by `SetupBinTable`):
- XEX2: `=SORT(UNIQUE(TOCOL(IF((A1:AO33<>1)*(A1:AO33<>2)*(A1:AO33<>""),A1:AO33,1/0),3)))` — Extracts all non-wall, non-dock cell values as bin names
- XEY2: `=MAP(XEX2#, LAMBDA(b, SUMPRODUCT(($A$1:$AO$33=b)*COLUMN($A$1:$AO$33))))` — Finds column of each bin
- XEZ2: `=MAP(XEX2#, LAMBDA(b, SUMPRODUCT(($A$1:$AO$33=b)*ROW($A$1:$AO$33))))` — Finds row of each bin

**ABC Ranking Thresholds** (calculated by `Analyze_Map_Locations`):
- **A** (closest): Distance <= 20th percentile
- **B** (medium): Distance <= 50th percentile
- **C** (farthest): Distance > 50th percentile

Color coding: A = green (RGB 198,239,206), B = yellow (RGB 255,235,156), C = red (RGB 255,199,206). Applied to both the rank cells and the corresponding Map_Grid cells.

### Bin Naming Convention

Format: `{Aisle}-{Rack}-{Level}-{Side}`

Example: `A3-R2-L2-A`
- A3 = Aisle 3
- R2 = Rack 2
- L2 = Level 2
- A = Side A

## Sheet 4: Transaction_History

**Purpose**: Permanent, append-only log of every confirmed operation. Used by `RebuildInventoryState` to replay and calculate current inventory.

| Column | Header | Data Type | Description |
|--------|--------|-----------|-------------|
| A | Order ID | Number | Sequential ID shared across all lines in one order |
| B | Date | Date | Transaction date |
| C | Time | Time | Transaction time (hh:mm:ss) |
| D | SKU | Text | Item identifier |
| E | Qty | Number | Quantity picked or put away |
| F | New Stock Level | Number | Snapshot of bin stock after this transaction |
| G | Type | Text | "PICK" or "PUTAWAY" |
| H | Distance from Start | Number | A* distance (m) from dock to this bin |
| I | Total Order Distance | Number | Total batch route distance for the entire order |
| J | Bin Location | Text | Bin name where operation occurred |
| K | Bin Rank | Text | ABC rank of the bin at time of transaction |

## VBA Module Structure (Module1.bas)

All code is in a single standard module organized into 6 sections.

### MOD 1: Globals & Constants

| Name | Type | Value | Purpose |
|------|------|-------|---------|
| `TILE_SCALE` | Public Const Double | 0.3848 | Meters per grid cell (converts steps to real distance) |
| `OpenList` | Public Object (Dictionary) | — | A* open set |
| `CameFrom` | Public Object (Dictionary) | — | A* parent tracking for path reconstruction |
| `gScore` | Public Object (Dictionary) | — | A* cost-from-start |
| `fScore` | Public Object (Dictionary) | — | A* estimated total cost (G + H) |

### MOD 2: Pathfinding Core (A*)

| Function/Sub | Parameters | Returns | Description |
|-------------|------------|---------|-------------|
| `FindPath` | StartStr, EndStr (e.g. "5,10") | Collection of "x,y" strings | A* shortest path between two grid coordinates. Uses Manhattan distance heuristic. Avoids cells with value "1". |
| `ProcessNeighbor` | nX, nY, Current, eX, eY, ws | — | Evaluates one neighbor cell (up/down/left/right) and updates open list if path improves. |
| `ReconstructPath` | CameFrom dict, Current | Collection | Traces parent chain backwards from goal to start, returns path in forward order. |
| `IsWall` | ws, X, Y | Boolean | Returns True if cell is out of bounds, equals 1, or contains "1". Value "2" (dock) is not a wall. |

### Math Utils

| Function/Sub | Description |
|-------------|-------------|
| `ReverseSegment` | Reverses a segment of parallel arrays (SKU, X, Y) in-place. Used by 2-OPT. |
| `ReverseSegmentCollection` | Reverses a segment of a Collection array in-place. |
| `GetGradientColor` | Maps a 0.0-1.0 ratio to a green-to-red RGB gradient for path coloring. |

### MOD 3: Map Helpers

| Function/Sub | Parameters | Returns | Description |
|-------------|------------|---------|-------------|
| `FindBestEmptyBin` | PreferredRank, RequiredQty, ExcludeData, TargetSKU | Bin name or "FULL" | Finds the closest empty bin of the requested rank that fits the SKU capacity. Uses Manhattan distance from dock. Excludes bins already allocated in the current session. |
| `GetAvailabilityStats` | wsHelper | String | Returns count of empty bins per rank: "Empty: A=X, B=Y, C=Z". |
| `FindPartialBin` | TargetSKU, wsHelper | "BinName\|Space" or "" | Finds a bin already holding this SKU with remaining space. Uses best-fit strategy (smallest remaining space). |
| `FindBinForSKU` | TargetSKU, ValQty | Bin name or "" | Finds a bin containing enough of the target SKU to fulfill a pick. |
| `GetSKUCapacity` | TargetSKU, wsHelper | Double (-1 if not found) | VLOOKUP on XEU:XEV to get the max capacity for a SKU. |
| `RegisterSKU` | TargetSKU, Capacity, wsHelper | — | Appends a new SKU and its capacity to the XEU:XEV registry. |
| `GetNextOrderID` | ws | Long | Returns MAX(Column A) + 1. Defaults to 1000 if empty. |

### MOD 4: Inventory State Engine

| Sub | Description |
|-----|-------------|
| `RebuildInventoryState` | **The core state engine.** Resets all bins to EMPTY/0, then replays every row in Transaction_History chronologically. PUTAWAY adds qty and sets SKU. PICK subtracts qty and clears SKU if qty reaches 0. Writes final state to Map_Helper columns XFC (Current SKU) and XFD (Current Qty). |

**Data flow**: Transaction_History (cols D,E,G,J) -> Dictionary replay -> Map_Helper (XFC, XFD)

### MOD 5: Operations & Logging

| Sub | Description |
|-----|-------------|
| `ConfirmOperations` | **Main user-facing macro.** Reads Form rows, registers unknown SKUs (prompts for capacity), validates putaway quantities against bin capacity, auto-assigns empty bins for putaways (with consolidation logic via `FindPartialBin`) and finds bins for picks, logs each line to Transaction_History, rebuilds inventory state, and optionally generates the map. |
| `LogTransaction` | Writes one row to Transaction_History with all 11 columns (Order ID, Date, Time, SKU, Qty, New Stock Level, Type, Distance from Start, Total Order Distance, Bin Location, Bin Rank). Looks up distance and current qty via XLOOKUP on Map_Helper. |
| `Clear_Form` | Clears Form data (rows 2+) and calls `ClearMap` to remove all generated shapes from Map_Grid. |

**ConfirmOperations Flow**:
1. Register any unknown SKUs (prompt user for capacity)
2. Assign Order ID and confirm with user
3. Validate: check bin capacity won't be exceeded
4. Auto-assign bins: try consolidation first (`FindPartialBin`), then find best empty bin by rank (`FindBestEmptyBin`). For picks, auto-find bin with enough stock (`FindBinForSKU`).
5. Verify all rows have a location
6. Calculate batch distance (`CalculateBatchDistance`)
7. Log each line to Transaction_History
8. Rebuild inventory state
9. Prompt to generate map visualization

### MOD 6: Mapping & Visualization Engine

| Sub/Function | Description |
|-------------|-------------|
| `GenerateMapPath` | **Main visualization macro.** Reads Form, resolves bin coordinates from Map_Helper, optimizes visit order with Nearest Neighbor + 2-OPT, runs A* pathfinding between each stop, draws colored path lines and target markers, draws stats box with total distance. Writes total distance to cell AQ1. |
| `ClearMap` | Removes cell coloring from A1:AO33 and deletes all shapes named `PathLine_Generated` or `PathLabel_Generated`. |
| `DrawGPSPath` | Draws arrow connectors between A* path steps with gradient coloring (green=start, red=end). Adds random jitter to prevent overlapping lines on revisited paths. |
| `DrawTargetMarker` | Places a shape (Oval/Triangle/Rectangle by level) at bin coordinates. Red=PICK, Green=PUTAWAY. Offsets when multiple markers share the same cell. |
| `DrawTinyLabel` | Adds a small text box with the SKU name next to a marker. |
| `DrawStatsBox` | Draws a dark rounded rectangle showing "TOTAL: X.Xm" in the top-right area of Map_Grid. |
| `CalculateBatchDistance` | Calculates total walking distance for the current Form order using NN + A* pathfinding (no visualization). Returns distance in meters. |
| `Analyze_Map_Locations` | Calculates round-trip A* distance for every bin, writes to XFA. Classifies bins into ABC ranks using 20th/50th percentile thresholds, writes to XFB. Color-codes both Map_Helper rank cells and Map_Grid floor cells. |
| `SetupBinTable` | Initializes Map_Helper headers and dynamic array formulas (SORT/UNIQUE/MAP/LAMBDA) that extract bin names and coordinates from Map_Grid. Run once when setting up a new map. |
| `SuggestEmptyBin` | User-facing tool. Prompts for a rank preference and shows the closest available empty bin. |
| `GetBinDistance` | XLOOKUP wrapper returning the pre-calculated distance (XFA) for a given bin name. |
| `VisualizeBin` | Places a marker on Map_Grid for a single bin by looking up its coordinates from Map_Helper. |

## Data Flow Summary

```
User fills Form (Sheet 1)
        |
        v
ConfirmOperations (VBA)
   |-- Validates & auto-assigns bins using Map_Helper data
   |-- Logs to Transaction_History (Sheet 4)
   |-- Calls RebuildInventoryState
   |       |
   |       v
   |   Replays Transaction_History -> Updates Map_Helper XFC/XFD
   |
   |-- Optionally calls GenerateMapPath
           |
           v
       Reads Form + Map_Helper coordinates
       NN + 2-OPT route optimization
       A* pathfinding on Map_Grid (Sheet 2)
       Draws path, markers, and stats on Map_Grid
```

## Setup Sequence (for a new warehouse)

1. Draw the warehouse layout in Map_Grid (A1:AO33): place "1" for walls, "2" for dock, bin names in storage cells
2. Run `SetupBinTable` to populate Map_Helper formulas
3. Run `Analyze_Map_Locations` to calculate distances and ABC ranks
4. Start entering orders in Form and running `ConfirmOperations`
